What are Arrays and Hashes in Ruby?

11-15-2015

First, understanding what each one is and how they're different is probably what I should cover. Arrays can be looked at as a list that contains a collection of data/information in some kind of order. Hashes can be looked at as a collection of pairs of data/information and each pair is used as a name(key) and it's value.

Now you might question when/why you would need to use one over the other, I'm going to help explain that. A lot of times accessing array elements is efficient, but hashes provide a much needed flexibility with custom keys. Arrays are ordered, integer-indexed collections of any object, which starts at 0 (You would access the first value of "array_name" with "array_name[0]"). Hashes on the other hand, can be defined with keys (name) and a value that can be accessed with their custom keys (i.e. "hash_name[key_name]" would return the value of "key_name"). View below for examples of both arrays and hashes.

Array Example: 

Array = ["Albert", 27]

Hash Example: 

Hash = { "Name" => "Albert",
      "Age" => 27
}
