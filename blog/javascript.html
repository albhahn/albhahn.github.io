JavaScript Objects Compared to Ruby Hashes!

12-06-2015

This week, we dove right into learning JavaScript. One of the first things I noticed, was that they are very similar, yet still different. An example of their similarities can be Javascript objects compared to Ruby hashes, but their differences will show in syntax.

In Ruby, we create a hash by simply declaring it a name and setting it to key/value pairs inside of braces ( { and } ). In Javascript, we create an object in a very similar way, but a main difference is instead of just declaring a name, we define it to a variable and use "properties" as the key/value pairs. Now one of the benefits JavaScript has over Ruby in this situation, is that you can have "functions" inside of properties. Below are examples of their syntax so you can have a visual image of both how similar and different they are (included a function as a property for Javascript).

Ruby: 

hash_name = {
  key_name: value,
  key2_name: second_value,
  key3_name: third_value
}

JavaScript: 

var objectName = {
  prop_name: value,
  prop2_name: second_value,
  functionName: function() {
    console.log("Functions actually do something");
  }
}
